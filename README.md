# Data Models Ebook

This book is designed to be read in order. Each chapter builds on the vocabulary, schemas, and case studies introduced in earlier chapters. Cross-references point back to the specific section where a concept was first developed.

## Hands-on Labs

Every chapter has a companion lab exercise available as a Google Colab notebook — no local installation required. The labs use the same datasets and scenarios as the chapter, so you can run the queries, break the constraints, and observe the failures yourself.

All labs are available at: [github.com/sreent/data-models-ebook](https://github.com/sreent/data-models-ebook)

| Chapter | Lab | Environment |
|---------|-----|-------------|
| 1 | Scenario reasoning | Markdown (discussion) |
| 2 | ERD to schema translation | Markdown (pen-and-paper) |
| 3 | Normalisation | Google Colab (SQL) |
| 4 | SQL querying | Google Colab (SQL) |
| 5 | Enforcement | Google Colab (SQL) |
| 6 | Document modelling | Google Colab (MongoDB) |
| 7 | XML contracts | Google Colab (Python/lxml) |
| 8 | RDF and knowledge graphs | Markdown (Turtle exercises) |
| 9 | SPARQL querying | Google Colab (rdflib) |
| 10 | Property graphs & Cypher | Google Colab (Neo4j) |
| 11 | Multi-model capstone | Markdown (design exercise) |

**Reading without the labs.** The book is self-contained — every query result and worked example is shown inline. You can read the entire book without running a single lab. The labs deepen understanding; they don't gate it.
